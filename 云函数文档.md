## getDashboardStats

> ⚠️ **已废弃**: 此云函数已废弃，请使用 `getInboundStats`。

获取入库统计数据，支持按日、周、月统计。

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| periodType | string | 否 | 统计周期：'day'(默认)、'week'、'month' |
| startDate | string | 否 | 开始日期 (YYYY-MM-DD)，仅 day 周期支持 |
| endDate | string | 否 | 结束日期 (YYYY-MM-DD)，仅 day 周期支持 |

### 返回结果

```javascript
{
  success: boolean,
  data: {
    chartData: Array<{period: string, count: number}>,
    totalInboundCount: number
  }
}
```

### 示例

```javascript
// 按日统计最近30天
wx.cloud.callFunction({
  name: 'getDashboardStats',
  data: { periodType: 'day' }
})

// 指定日期范围
wx.cloud.callFunction({
  name: 'getDashboardStats',
  data: {
    periodType: 'day',
    startDate: '2024-01-01',
    endDate: '2024-01-31'
  }
})

// 按周统计最近12周
wx.cloud.callFunction({
  name: 'getDashboardStats',
  data: { periodType: 'week' }
})

// 按月统计最近12个月
wx.cloud.callFunction({
  name: 'getDashboardStats',
  data: { periodType: 'month' }
})
```

```javascript
  name: 'getInboundStats',
  data: {
    periodType: 'day',
    startDate: '2024-01-01',

  }

```

## getOperationLogs

查询操作日志记录，支持多种筛选条件和分页查询。

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| operationType | string | 否 | 操作类型：'create'、'update'、'delete' |
| logType | string | 否 | 日志类型：'inbound'、'outbound' |
| logId | string | 否 | 关联的入库/出库记录ID |
| operator | string | 否 | 操作人昵称 |
| startDate | string | 否 | 开始日期 (YYYY-MM-DD) |
| endDate | string | 否 | 结束日期 (YYYY-MM-DD) |
| limit | number | 否 | 每页数量，默认20，最大100 |
| cursor | string | 否 | 游标，首次查询不传 |

### 返回结果

```javascript
{
  success: boolean,
  data: Array<{
    _id: string,
    operationType: string,
    logType: string,
    logId: string,
    operationContent: string,
    operator: string,
    operatorOpenid: string,
    operationTime: string,
    createTime: object
  }>,
  cursor: string | null,
  hasMore: boolean,
  total: number,
  errMsg: string
}
```

### 示例

```javascript
// 首次查询所有操作日志
wx.cloud.callFunction({
  name: 'getOperationLogs',
  data: { limit: 20 }
})

// 按操作类型筛选
wx.cloud.callFunction({
  name: 'getOperationLogs',
  data: {
    operationType: 'create',
    limit: 20
  }
})

// 按日志类型和日期范围筛选
wx.cloud.callFunction({
  name: 'getOperationLogs',
  data: {
    logType: 'inbound',
    startDate: '2024-01-01',
    endDate: '2024-01-31',
    limit: 20
  }
})

// 分页查询
wx.cloud.callFunction({
  name: 'getOperationLogs',
  data: {
    limit: 20,
    cursor: 'previous_cursor'
  }
})
```

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| action | string | 是 | 操作类型：'add'、'remove'、'list'、'check' |
| openid | string | 条件 | 用户openid（add、remove、check时必填） |
| remark | string | 否 | 备注（add时使用） |

### 返回结果

**add操作：**
```javascript
{
  success: boolean,
  _id: string,
  message: string
}
```

**remove操作：**
```javascript
{
  success: boolean,
  removed: number,
  message: string
}
```

**list操作：**
```javascript
{
  success: boolean,
  data: Array<{openid: string, remark: string, createTime: Date, ...}>
}
```

**check操作：**
```javascript
{
  success: boolean,
  isWhitelisted: boolean
}
```

### 示例

```javascript
// 添加用户到白名单

  name: 'manageWhitelist',

    action: 'add',
    openid: 'user_openid',

  }

  name: 'manageWhitelist',
  data: {
    action: 'remove',
    openid: 'user_openid'
  }
})

wx.cloud.callFunction({
  name: 'manageWhitelist',
  data: { action: 'list' }
})

// 检查用户是否在白名单
```

## queryRecords

查询出入库记录，支持游标分页和多种筛选条件。

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| type | string | 是 | 查询类型：'inbound' 或 'outbound' |
| limit | number | 否 | 每页数量，默认20，最大100 |
| cursor | string | 否 | 游标，首次查询不传 |
| customerName | string | 否 | 客户姓名（模糊查询） |
| trackingNumber | string | 否 | 物流单号（模糊查询） |
| outboundNumber | string | 否 | 出库编号（仅出库，模糊查询） |
| outboundStatus | string | 否 | 出库状态（仅出库）：`pending`、`completed`、`cancelled` |
| source | string | 否 | 出库来源（仅出库）：`order`、`manual` |
| shopName | string | 否 | 店铺名称（仅入库，模糊查询） |
| channelType | string | 否 | 渠道类型（仅入库）：'return'、'afterSale'、'purchase'、'recycle' |
| model | string | 否 | 手机型号（模糊查询） |
| hasIssue | boolean | 否 | 是否有损坏/配件不全（仅入库）：true=有，false=正常或不包含此字段 |
| startDate | string | 否 | 开始日期 (YYYY-MM-DD) |
| endDate | string | 否 | 结束日期 (YYYY-MM-DD) |

### 返回结果

```javascript
{
  success: boolean,
  data: Array<{...}>,
  cursor: string | null,
  hasMore: boolean,
  errMsg: string
}
```

### 示例

```javascript
// 首次查询入库记录
wx.cloud.callFunction({
  name: 'queryRecords',
  data: {
    data: {
      type: 'inbound',
      limit: 20,
      customerName: '张三',
      shopName: '某某采购',
      channelType: 'purchase',
      model: 'iPhone 14',
      startDate: '2024-01-01',
      endDate: '2024-01-31'
    }
  }
}).then(res => {
  console.log(res.result.data)
  if (res.result.hasMore) {
    // 加载下一页
    return wx.cloud.callFunction({
      name: 'queryRecords',
      data: {
        data: {
          type: 'inbound',
          limit: 20,
          cursor: res.result.cursor
        }
      }
    })
  }
})

// 查询有损坏/配件不全的入库记录
wx.cloud.callFunction({
  name: 'queryRecords',
  data: {
    data: {
      type: 'inbound',
      hasIssue: true,
      limit: 20
    }
  }
})

// 查询正常的入库记录（包括没有 hasIssue 字段的旧记录）
wx.cloud.callFunction({
  name: 'queryRecords',
  data: {
    data: {
      type: 'inbound',
      hasIssue: false,
      limit: 20
    }
  }
})
```

## saveOrders

批量保存订单记录到数据库，用于 Excel 导入场景。

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| orders | Array | 是 | 订单数据数组，每项为 OrderRecord 对象（不含 _id） |

### orders 每项字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| serialNumber | number | 是 | 序号 |
| date | string | 是 | 日期 |

## generateOutboundRecord

根据订单生成一条 `pending` 待出库单，并回写订单的出库关联字段。当前用于订单页面「生成出库」按钮。

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| orderId | string | 是 | 订单 `_id` |

### 生成条件

| 条件 | 说明 |
|------|------|
| 订单存在 | 从 `orders` 集合读取订单 |
| 订单未关联出库单 | `linkedOutboundId` / `linkedOutboundNumber` 为空 |
| 订单处于待发货状态 | `status` 为空、`unknown` 或 `--` |
| 订单需要实物出库 | 排除 `noShip`、虚拟产品、仅退款、退租金等无需实物出库场景 |

### 落库行为

使用数据库事务保证以下操作原子性：

1. 读取 `orders`
2. 通过 `system_counters` 生成 `CK-YYYYMMDD-NNNNN` 出库编号
3. 写入 `outbound_records`，状态为 `pending`
4. 回写 `orders.linkedOutboundId`、`orders.linkedOutboundNumber`、`orders.outboundSyncStatus = pending`

### 返回结果

```javascript
{
  success: boolean,
  data: {
    outboundId: string,
    outboundNumber: string,
    outboundStatus: 'pending'
  },
  errMsg: string
}
```

### 示例

```javascript
wx.cloud.callFunction({
  name: 'generateOutboundRecord',
  data: {
    data: {
      orderId: 'order_id'
    }
  }
})
```

### 注意事项

1. 如果订单已有关联出库单，函数会幂等返回成功并返回已有出库信息。
2. 如果订单不是待发货状态，返回失败，不生成出库单。
3. 该函数不修改订单发货状态；实际发货由 `completeOutbound` 完成。

## completeOutbound

完成出库单，并将物流单号回写到关联订单。当前用于小程序/出库确认场景。

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| outboundId | string | 是 | 出库单 `_id` |
| trackingNumber | string | 是 | 物流单号 |
| completedBy | string | 否 | 出库操作人 |
| phoneModels | Array | 否 | 实际出库货品明细，可修正数量和型号 |

### 落库行为

使用数据库事务保证以下操作原子性：

1. 读取 `outbound_records`
2. 校验当前出库状态为 `pending`
3. 更新 `outbound_records.outboundStatus = completed`
4. 写入 `trackingNumber`、`completedDate`、`completedBy`
5. 回写关联订单 `orders.trackingNumber`、`orders.status = shipped`、`orders.outboundSyncStatus = completed`
6. 写入 `operation_logs`

### 返回结果

```javascript
{
  success: boolean,
  data: {
    outboundId: string,
    linkedOrderId: string,
    trackingNumber: string,
    outboundStatus: 'completed',
    orderUpdated: boolean,
    orderMissing?: boolean,
    idempotent?: boolean
  },
  errMsg: string
}
```

### 示例

```javascript
wx.cloud.callFunction({
  name: 'completeOutbound',
  data: {
    data: {
      outboundId: 'outbound_record_id',
      trackingNumber: 'SF123456789',
      completedBy: '出库同事'
    }
  }
})
```

### 幂等与异常

| 场景 | 处理 |
|------|------|
| 重复提交相同物流号 | 返回成功，`idempotent = true` |
| 已完成但物流号不同 | 返回失败，避免覆盖已完成记录 |
| 关联订单不存在 | 仍完成出库，`linkedOrderStatus = missing`，返回 `orderMissing = true` |
| 关联订单已发货且物流号不同 | 返回失败，避免订单物流号被错误覆盖 |

## cancelOutbound

取消订单关联的待出库单。当前用于订单页面「取消出库」按钮。

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| orderId | string | 是 | 订单 `_id` |
| reason | string | 否 | 取消原因，默认“手动取消出库” |
| operator | string | 否 | 操作人 |

### 落库行为

使用数据库事务保证以下操作原子性：

1. 读取 `orders`
2. 读取订单关联的 `outbound_records`
3. 校验出库单未完成
4. 更新 `outbound_records.outboundStatus = cancelled`
5. 写入 `cancelledDate`、`cancelledBy`、`cancelReason`
6. 清空 `orders.linkedOutboundId`、`orders.linkedOutboundNumber`
7. 设置 `orders.outboundSyncStatus = none`
8. 写入 `operation_logs`

### 返回结果

```javascript
{
  success: boolean,
  data: {
    outboundId: string,
    outboundNumber: string,
    outboundStatus: 'cancelled',
    orderUpdated: boolean,
    outboundMissing?: boolean,
    idempotent?: boolean
  },
  errMsg: string
}
```

### 示例

```javascript
wx.cloud.callFunction({
  name: 'cancelOutbound',
  data: {
    data: {
      orderId: 'order_id',
      reason: '订单信息变更，暂不出库',
      operator: '管理员'
    }
  }
})
```

### 注意事项

1. 已完成出库单不能取消。
2. 如果订单没有关联出库单，函数会幂等返回成功。
3. 如果订单关联的出库单已不存在，会清空订单出库关联，避免订单卡在“已生成”状态。

```javascript
wx.cloud.callFunction({

  }
```

### addModels - 为品牌添加型号

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| action | string | 是 | 固定值 'addModels' |
| brand | string | 是 | 品牌名称 |
| models | string[] | 是 | 要添加的型号数组 |

**返回结果：**

```javascript
{
  success: boolean,
  errMsg: string,
  addedCount: number  // 实际添加的型号数量
}
```

**示例：**

```javascript
wx.cloud.callFunction({
  name: 'phoneModels',
  data: {
    action: 'addModels',
    brand: 'Apple',
    models: ['iPhone 17', 'iPhone 17 Pro']
  }
}).then(res => {
  if (res.result.success) {
    console.log(res.result.errMsg)  // 成功添加 2 个型号
    console.log('添加数量:', res.result.addedCount)
  } else {
    console.log(res.result.errMsg)  // 品牌不存在
  }
})
```

**注意事项：**
- 如果型号已存在，会自动去重，不会重复添加
- 如果所有型号都已存在，返回成功但 `addedCount` 为 0
- 品牌必须先通过 `addBrand` 创建，否则返回"品牌不存在"

## 数据库集合说明

### inbound_records
入库记录集合，存储所有入库相关信息。

### orders
订单记录集合，存储订单明细数据，字段对齐 Excel「订单明细」工作表，并额外保存订单-出库关联字段：

| 字段 | 说明 |
|------|------|
| linkedOutboundId | 关联出库单 `_id` |
| linkedOutboundNumber | 关联出库编号 |
| outboundSyncStatus | 出库同步状态：`none`、`pending`、`completed` |

### outbound_records
出库单生命周期集合，存储待出库、已出库、已取消的出库单。

| 字段 | 说明 |
|------|------|
| outboundNumber | 出库编号，如 `CK-20260621-00001` |
| outboundStatus | 出库状态：`pending`、`completed`、`cancelled` |
| linkedOrderId | 关联订单 `_id` |
| linkedOrderSerialNumber | 关联订单序号 |
| linkedOrderStatus | 关联订单状态：`active`、`deleted`、`missing` |
| completedDate / completedBy | 实际出库完成时间和操作人 |
| cancelledDate / cancelledBy / cancelReason | 取消时间、操作人和原因 |
| source | 来源：`order`、`manual` |

### system_counters
系统计数器集合。出库编号使用 `db_counter_outbound_YYYYMMDD` 计数器生成每日流水号。

### shops
店铺信息集合，存储渠道/店铺数据。

### user_whitelist
用户白名单集合，存储有权限访问的用户openid，包含昵称和头像信息。

### phone_models
手机型号集合，存储手机品牌和型号信息。

### operation_logs
操作日志集合，存储所有操作记录（入库、出库等）。

### record_history
修改历史集合，存储记录的所有修改历史，包含修改人、修改时间、变更字段等信息。

## 注意事项

1. 所有云函数都使用 `wx-server-sdk` 和微信云开发环境
2. 返回结果统一包含 `success` 字段表示操作是否成功
3. `queryRecords` 和 `getOperationLogs` 使用游标分页方式（基于 `skip`），适合大量数据查询
4. 图片URL通过 `getRealImageUrl` 获取真实可访问的临时链接
5. 用户权限通过白名单机制控制，在 `login` 云函数中验证
6. `login` 云函数支持更新用户的昵称和头像信息
7. `getDashboardStats` 已废弃，请使用 `getInboundStats` 获取入库统计数据
8. `getInboundStats` 支持按日统计入库记录数和手机台数
9. `getOutboundStats` 支持按日统计数据
10. 操作日志通过 `saveOperationLog` 和 `getOperationLogs` 云函数管理，支持增删查记录
11. `operation_logs` 集合存储所有操作记录，包含操作类型、日志类型、操作人等信息
12. 订单-出库联动使用 `outbound_records.outboundStatus` 表示生命周期：`pending`（待出库）、`completed`（已出库）、`cancelled`（已取消）
13. `generateOutboundRecord`、`completeOutbound`、`cancelOutbound` 均使用数据库事务保证订单和出库单一致性
14. `completeOutbound` 对相同物流号重复提交做幂等处理，不同物流号会拒绝覆盖已完成出库单
15. `cancelOutbound` 只允许取消未完成出库单，已完成出库单不能取消
