# 订单字段关联关系说明

本文档基于当前订单管理页面逻辑整理，覆盖订单新增、编辑、筛选、发货、申请快递、导出等流程中的字段动态关联关系。

## 1. 订单状态

订单状态字段为 `status`，字典来源于 `ORDER_STATUS_MAP`。

| 状态值 | 展示文本 | 说明 |
| --- | --- | --- |
| `shipped` | 已发货 | 已有物流信息的订单 |
| `noShip` | 不用发货 | 不需要发货的订单 |
| `returnReceived` | 退货已收 | 退货已收到 |
| `returnShipped` | 退发已发 | 退发已发出 |
| `unknown` | -- | 待处理/未明确状态 |

### 1.1 状态与物流字段

| 触发条件 | 关联字段 | 动态行为 |
| --- | --- | --- |
| `status === shipped` | `shippingFee` | 显示并要求填写邮寄结算方式 |
| `status === shipped` | `trackingNumber` | 显示并要求填写物流单号 |
| `status !== shipped` | `shippingFee` | 切换状态时自动清空 |
| `status !== shipped` | `trackingNumber` | 切换状态时自动清空 |

### 1.2 状态与操作按钮

| 触发条件 | 操作 | 动态行为 |
| --- | --- | --- |
| `status === unknown` 或旧数据为 `--` | 发货 | 显示发货按钮 |
| `status === unknown` 或旧数据为 `--` | 申请快递 | 显示申请快递按钮 |
| 其他状态 | 发货 | 不显示发货按钮 |
| 其他状态 | 申请快递 | 不显示申请快递按钮 |

### 1.3 发货流程写回

用户点击发货后，系统使用订单的 `consignee` 去匹配出库记录的 `customerName`。

| 步骤 | 字段 | 行为 |
| --- | --- | --- |
| 查询发货记录 | `consignee` | 作为客户名称关键词查询出库记录 |
| 发货记录列表 | `outboundDate` | 展示为发货时间 |
| 发货记录列表 | `customerName` | 展示为客户名称 |
| 发货记录列表 | `trackingNumber` | 展示为发货单号 |
| 用户选择记录 | `selectedShipRecord` | 打开确认发货弹窗，不立即写数据库 |
| 确认发货 | `shippingFee` | 用户选择包邮或到付 |
| 确认发货 | `trackingNumber` | 写入选中发货记录的快递单号 |
| 确认发货 | `status` | 写入 `shipped` |
| 确认发货 | `shippingFee` | 写入用户选择的 `prepaid` 或 `cod` |

### 1.4 申请顺丰快递写回

顺丰云函数 `applySfExpress` 只允许订单状态为 `--` 的订单申请快递。

| 条件/结果 | 关联字段 | 行为 |
| --- | --- | --- |
| 申请前校验 | `status` | 只允许 `unknown` 或 `--` |
| 申请前校验 | `trackingNumber` | 已有物流单号则拒绝重复申请 |
| 申请前校验 | `sfWaybillNo` | 已有顺丰单号则拒绝重复申请 |
| 申请前校验 | `consignee` | 收货人名称必填 |
| 申请前校验 | `consigneePhone` | 收货人电话必填 |
| 申请前校验 | `consigneeAddress` | 收货人地址必填 |
| 申请前校验 | `productName` | 货品名称必填 |
| 顺丰下单成功 | `status` | 写入 `shipped` |
| 顺丰下单成功 | `trackingNumber` | 写入顺丰返回运单号 |
| 顺丰下单成功 | `shippingFee` | 使用订单原值，否则默认 `prepaid` |
| 顺丰下单成功 | `expressProvider` | 写入 `sf` |
| 顺丰下单成功 | `expressApplyStatus` | 写入 `applied` |
| 顺丰下单成功 | `sfWaybillNo` | 写入顺丰运单号 |

## 2. 订单来源、订单类型、订单属性

### 2.1 订单来源与订单类型

订单来源字段为 `orderSource`，订单类型字段为 `orderType`。

| 订单来源 | 允许订单类型 |
| --- | --- |
| `new` 新增 | `newBusiness` 新增业务 |
| `service` 服务 | `postRentalShip` 租后发货、`postRentalReturn` 租后退货、`postRentalPayment` 租后款项、`deposit` 押金 |

动态行为：

| 触发条件 | 关联字段 | 动态行为 |
| --- | --- | --- |
| 切换 `orderSource` | `orderType` | 如果当前订单类型不在新来源允许范围内，自动清空订单类型 |
| 未选择 `orderSource` | `orderType` | 显示全部订单类型选项 |

### 2.2 订单类型与虚拟产品

部分订单类型会限制虚拟产品的货品名称。

| 订单类型 | 虚拟产品允许货品名称 |
| --- | --- |
| `newBusiness` 新增业务 | 平台租金、续期租金 |
| `postRentalPayment` 租后款项 | 补收差价、仅退款、维修费、快递费 |
| `deposit` 押金 | 收押金、退押金 |

动态行为：

| 触发条件 | 关联字段 | 动态行为 |
| --- | --- | --- |
| 切换 `orderType` | `products[].productName` | 如果是租赁1且当前虚拟产品货品名称不在白名单内，自动清空货品名称和规格 |
| `orderType=postRentalShip` 或 `postRentalReturn` | `returnStatus` | 显示并要求填写归还状态 |

### 2.3 订单属性与金额字段

订单属性字段为 `orderAttribute`。

| 触发条件 | 关联字段 | 动态行为 |
| --- | --- | --- |
| `orderAttribute === rental1` 且 `brand !== 虚拟产品` | `unitPrice` | 不显示、不要求填写 |
| `orderAttribute === rental1` 且 `brand !== 虚拟产品` | `amount` | 不显示、不要求填写 |
| `orderAttribute === rental1` 且 `brand !== 虚拟产品` | `paymentAccount` | 不显示、不要求填写 |
| 其他情况 | `unitPrice` | 显示并要求填写 |
| 其他情况 | `amount` | 根据数量和单价自动计算 |
| 其他情况 | `paymentAccount` | 显示并要求填写 |

## 3. 销售渠道与渠道类别

销售渠道字段为 `salesChannel`，渠道类别字段为 `channelCategory`。

系统通过 `salesChannel` 自动计算 `channelCategory`。

| 渠道类别 | 说明 |
| --- | --- |
| `platform` 平台 | 平台渠道 |
| `offline` 线下 | 非平台渠道 |

动态行为：

| 触发条件 | 关联字段 | 动态行为 |
| --- | --- | --- |
| 切换 `salesChannel` | `channelCategory` | 自动计算渠道类别 |
| `channelCategory === platform` | `onlineOrderNumber` | 显示并要求填写网店订单号 |
| `channelCategory === offline` | `onlineOrderNumber` | 自动清空网店订单号 |

## 4. 货品字段

货品字段保存在 `products` 数组中。

| 字段 | 关联字段 | 动态行为 |
| --- | --- | --- |
| `products[].brand` | `products[].productName` | 切换品牌时清空货品名称和规格 |
| `products[].productName` | `products[].specification` | 切换货品名称时清空规格 |
| `products[].quantity` | `products[].amount` | 数量变化时自动计算金额 |
| `products[].unitPrice` | `products[].amount` | 单价变化时自动计算金额 |
| `products[].amount` | `products[].paymentAccount` | 非租赁1普通货品场景下，金额字段相关记录要求选择收款账户 |

计算公式：

```txt
amount = quantity * unitPrice
```

新增或编辑订单时至少需要一条货品记录。

## 5. 转租赁2字段

转租赁2字段保存在 `transferProducts` 数组中。

### 5.1 显示条件

| 触发条件 | 关联字段 | 动态行为 |
| --- | --- | --- |
| 任一 `products[].productName === 部分转租赁2` | `transferProducts` | 显示转租赁2信息区块 |
| 任一 `products[].productName === 全部转租赁2` | `transferProducts` | 显示转租赁2信息区块 |
| 显示转租赁2且 `transferProducts` 为空 | `transferProducts` | 自动添加一组默认转租赁2记录 |

### 5.2 字段联动

| 字段 | 关联字段 | 动态行为 |
| --- | --- | --- |
| `transferProducts[].brand` | `transferProducts[].productName` | 切换品牌时清空货品名称和规格 |
| `transferProducts[].productName` | `transferProducts[].specification` | 切换货品名称时清空规格 |
| 出现转租赁2货品 | `paidPeriod` | 要求填写已交租期 |
| 出现转租赁2货品 | `paidRent` | 要求填写已交租金 |

保存时，系统会把 `transferProducts` 序列化到 `transferItems`，同时兼容旧字段：

| 新结构 | 旧兼容字段 |
| --- | --- |
| `transferProducts[0].brand` | `transferBrand` |
| `transferProducts[0].productName` | `transferProductName` |
| `transferProducts[0].specification` | `transferSpecification` |
| `transferProducts[0].paidPeriod` | `paidPeriod` |
| `transferProducts[0].paidRent` | `paidRent` |
| `transferProducts` 全量 | `transferItems` JSON 字符串 |

## 6. 收件人字段

| 字段 | 关联逻辑 |
| --- | --- |
| `consignee` | 发货操作用它查询出库记录的客户名称 |
| `consignee` | 顺丰申请快递时必填 |
| `consigneePhone` | 顺丰申请快递时必填 |
| `consigneeAddress` | 顺丰申请快递时必填 |
| 粘贴识别文本 | 可自动填入 `consignee`、`consigneePhone`、`consigneeAddress` |

## 7. 发货记录匹配字段

发货记录来自出库记录，查询条件为：

| 订单字段 | 发货记录字段 | 说明 |
| --- | --- | --- |
| `consignee` | `customerName` | 使用收件人名称匹配发货记录客户名称 |

发货记录列表展示：

| 展示行 | 发货记录字段 |
| --- | --- |
| 发货时间 | `outboundDate` |
| 客户名称 | `customerName` |
| 发货单号 | `trackingNumber` |

用户选择某条发货记录并确认后写回：

| 订单字段 | 写入值 |
| --- | --- |
| `trackingNumber` | 选中发货记录的 `trackingNumber` |
| `status` | `shipped` |
| `shippingFee` | 用户在确认弹窗选择的 `prepaid` 或 `cod` |

## 8. 归还状态字段

归还状态字段只在部分订单类型下显示。

| 触发条件 | 关联字段 | 动态行为 |
| --- | --- | --- |
| `orderType === postRentalShip` | `returnStatus` | 显示并要求填写 |
| `orderType === postRentalReturn` | `returnStatus` | 显示并要求填写 |
| `returnStatus === inTransit` | `returnTrackingNumbers` | 显示并要求填写归还物流单号 |
| `returnStatus !== inTransit` | `returnTrackingNumbers` | 不要求填写 |

归还状态字典：

| 状态值 | 展示文本 |
| --- | --- |
| `returned` | 产品已退回入库 |
| `inTransit` | 产品运输途中 |
| `notReturned` | 客户未退回 |

## 9. 附件字段

| 字段 | 动态行为 |
| --- | --- |
| `attachments` | 编辑订单时显示已有附件 |
| 新上传附件 | 保存时上传到云存储后追加到 `attachments` |
| 删除已有附件 | 从 `attachments` 中移除 |

## 10. 筛选字段

订单列表筛选条件与数据库字段关系如下。

| 筛选项 | 对应字段 | 查询方式 |
| --- | --- | --- |
| 网店订单号 | `onlineOrderNumber` | 模糊匹配 |
| 客户名称 | `customerName` | 模糊匹配 |
| 人员 | `salesperson` | 精确匹配 |
| 订单类型 | `orderType` | 精确匹配 |
| 订单状态 | `status` | 精确匹配 |
| 日期 | `date` | 按开始日期/日期范围筛选 |

## 11. 导出字段

导出流程使用独立筛选条件。

| 导出条件 | 对应字段 | 说明 |
| --- | --- | --- |
| 开始日期 | `date` | 必填 |
| 结束日期 | `date` | 必填 |
| 日期范围 | `date` | 不能超过 183 天 |
| 销售渠道多选 | `salesChannel` | 多选时逐渠道查询并合并 |
| 人员多选 | `salesperson` | 多选时逐人员查询并合并 |

导出时会按 `_id` 去重，避免多条件组合查询导致重复记录。

## 12. 保存展开规则

新增或编辑订单时，前端支持一个订单里维护多条货品，但保存时会展开成多条订单记录。

| 表单结构 | 保存结果 |
| --- | --- |
| `products[0]` | 更新或新增主订单 |
| `products[1...]` | 额外新增多条订单记录 |
| 公共字段 | 复制到每一条展开记录 |
| 每条货品字段 | 分别写入对应展开记录 |

公共字段包括：

```txt
serialNumber, date, orderSource, orderAttribute, orderType,
salesChannel, salesperson, channelCategory, onlineOrderNumber,
customerName, consignee, consigneePhone, consigneeAddress,
shippingFee, status, customerRemark, transferItems,
attachments, returnStatus, returnTrackingNumbers
```

## 13. 操作按钮关联关系

| 按钮 | 显示条件 | 动态行为 |
| --- | --- | --- |
| 详情 | 始终显示 | 打开订单详情 |
| 编辑 | 始终显示 | 打开 6 步编辑向导 |
| 删除 | 始终显示 | 打开删除确认 |
| 发货 | `status === unknown` 或旧数据 `--` | 查询发货记录，二次确认后写回发货信息 |
| 申请快递 | `status === unknown` 或旧数据 `--` | 调用顺丰下单云函数 |

## 14. 核心关系摘要

```txt
orderSource -> orderType 可选范围
salesChannel -> channelCategory -> onlineOrderNumber 是否显示/必填
orderAttribute + brand -> unitPrice/amount/paymentAccount 是否显示/必填
brand -> productName -> specification
quantity + unitPrice -> amount
productName=部分转租赁2/全部转租赁2 -> transferProducts
orderType=租后发货/租后退货 -> returnStatus
returnStatus=运输途中 -> returnTrackingNumbers
status=已发货 -> shippingFee + trackingNumber
status=-- -> 发货/申请快递按钮
consignee -> 匹配出库记录 customerName -> trackingNumber
```
